# 🔐 扫码登录使用指南

**重要**: 本系统采用**扫码登录**方式，无需手动填写任何API密钥！

---

## ✨ 为什么选择扫码登录?

| 方式 | 安全性 | 易用性 | 推荐度 |
|------|--------|--------|--------|
| 手动API密钥 | ⚠️ 中等 | ❌ 复杂 | ❌ 不推荐 |
| **扫码登录** | ✅ 极高 | ✅ 极简 | ✅ **推荐** |

**扫码登录的优势:**
- ✅ **0风险**: API密钥永远不会存储在本地
- ✅ **一键登录**: 打开应用→扫码→完成，只需10秒
- ✅ **自动续期**: 令牌自动刷新，无需手动管理
- ✅ **完全可控**: 随时可在Hyperliquid官方app中撤销权限
- ✅ **审计追踪**: 所有操作都有完整的审计日志

## 🚀 使用流程

### 第一次启动应用

```
1️⃣ 启动应用
   docker-compose up -d

2️⃣ 打开浏览器访问

   应用自动获得权限并跳转到交易界面
   整个过程不到 10 秒！
```

## 🔒 安全细节

### 1. 零API密钥存储

```
传统方式 (不安全):
API密钥 → 本地文件 → 可能泄露

扫码登录 (安全):
二维码 → Hyperliquid服务器 → OAuth令牌 → 临时会话
(API密钥永远不会传输或存储)
```

### 2. OAuth 2.0 标准

- ✅ 使用行业标准的 OAuth 2.0 协议
- ✅ 所有通信都是加密的 (HTTPS)
- ✅ 令牌有时间限制，自动过期
- ✅ 可随时在Hyperliquid官方app中撤销

### 3. 权限隔离

应用只获得必要的权限:
- ✅ 读取账户和持仓信息
- ✅ 查看订单历史
- ✅ 执行交易
- ❌ 无法修改账户设置
- ❌ 无法修改安全设置
- ❌ 无法提现

### 4. 完整的审计日志

所有操作都记录:
- 登录时间
- 交易记录
- 系统状态
- 错误和异常

## 🆘 常见问题

### Q: 二维码无法扫描?

**A:** 试试这些方法:
1. 刷新页面 (Ctrl+R 或 Cmd+R)
2. 检查浏览器是否已打开 http://localhost:3000
3. 检查后端是否正在运行 (查看docker日志)
4. 检查网络连接

如果还是不行，运行:
```bash
# 检查API状态
curl http://localhost:8000/api/auth/generate-qr

# 查看错误日志
docker-compose logs api | tail -50
```

### Q: 扫码后没有反应?

**A:** 
1. 确保在Hyperliquid App中完成了授权
2. 等待5秒让应用处理
3. 如果界面仍未跳转，试试点击"刷新QR码"
4. 重新扫码

### Q: 可以多台设备同时登录吗?

**A:** 
- ✅ 可以，每次扫码都会生成新的会话
- ✅ 旧会话仍然有效
- ✅ 支持多设备同时运行

### Q: 登录后多久会过期?

**A:**
- ✅ 访问令牌有效期: 24小时
- ✅ 会话有效期: 30天
- ✅ 自动刷新，无需重新登录
- ✅ 如果30天未使用，需要重新登录

### Q: 如何撤销权限?

**A:**
在 Hyperliquid 官方 App 中:
1. 进入"用户中心" → "授权应用"
2. 找到"Hyperliquid AI Trader"
3. 点击"撤销权限"

权限撤销后应用立即失效。

### Q: 支持沙箱环境吗?

**A:**
✅ 支持！修改 .env:
```
HYPERLIQUID_SANDBOX_MODE=true
```

使用沙箱二维码，所有交易都是虚拟的。
