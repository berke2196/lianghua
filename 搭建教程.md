# AsterDex HFT 搭建教程

> 本教程适用于在 Windows 本地环境部署后端，前端托管在 GitHub Pages（域名 xb1.me）

```
├── 启动全部.bat           # 一键启动前端+后端
└── .env                  # 环境变量（自动生成）
```

## 三、首次部署步骤

```bash
git clone https://github.com/berke2196/lianghua.git
cd lianghua
```

### 步骤 2：安装 Python 依赖

```bash
pip install -r requirements.txt
```

### 步骤 3：安装 Node 依赖


```bash
python -c "from db import init_db; init_db()"
```

双击 `启动全部.bat`，或分别运行：

```bash

python run.py

# 终端2：启动前端（开发模式）
npm start
```

## 五、前端部署到 GitHub Pages（xb1.me）

前端是纯静态 React 应用，托管在 GitHub Pages。

### 部署命令（每次更新前端后执行）

```bash
npm run deploy
```

这会自动：
1. 构建 React 生产包（`npm run build`）
2. 推送到 `gh-pages` 分支
3. GitHub 自动发布到 `https://xb1.me`

### DNS 配置（域名 xb1.me → GitHub Pages）

在域名服务商设置以下 DNS 记录：

## 六、后端部署到公网服务器（让 xb1.me 能连接）

**GitHub Pages 是纯静态，前端无法运行后端。**  
需要一台有公网 IP 的服务器（云服务器/VPS）来运行后端。

### 推荐方案：VPS + Nginx 反向代理

#### 1. 服务器上安装依赖

```bash
apt update && apt install -y python3 python3-pip nginx certbot python3-certbot-nginx
pip3 install -r requirements.txt
```

#### 2. 上传代码到服务器

```bash
git clone https://github.com/berke2196/lianghua.git /opt/asterdex
cd /opt/asterdex
```

#### 3. 配置环境变量

编辑 `/opt/asterdex/.env`：

```env
SECRET_KEY=你的随机密钥（至少32位）
ADMIN_PASSWORD=管理员密码
BACKEND_HOST=0.0.0.0
BACKEND_PORT=8000
FORCE_HTTPS=1
```

#### 4. 启动后端（后台运行）

```bash
nohup python3 run.py > logs/backend.log 2>&1 &
```

或用 systemd（推荐）：

```bash
cp asterdex.service /etc/systemd/system/
systemctl enable asterdex
systemctl start asterdex
```

#### 5. 配置 Nginx 反向代理（支持 HTTPS + WebSocket）

编辑 `/etc/nginx/sites-available/asterdex`：

```nginx
server {
    listen 80;
    server_name api.xb1.me;   # 后端用子域名

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

```bash
ln -s /etc/nginx/sites-available/asterdex /etc/nginx/sites-enabled/
nginx -t && systemctl reload nginx
certbot --nginx -d api.xb1.me   # 申请 SSL
```

#### 6. 前端指向后端服务器

在前端项目根目录创建 `.env.production`：

```env
REACT_APP_API_URL=https://api.xb1.me
```

然后重新部署前端：

```bash
npm run deploy
```

首次启动后，系统自动创建管理员账号：

## 九、常见问题

**Q: 前端显示"网络错误，无法连接服务器"**  
A: 后端未启动，或前端的 `REACT_APP_API_URL` 没有指向后端地址。

**Q: 登录后一直转圈**  
A: 检查后端日志，通常是 AsterDex API 连接超时。

**Q: gh-pages 部署失败**  
A: 确认 Git remote 已配置正确，且 Token 有效。重新运行 `npm run deploy`。

**Q: 后端启动报 `ModuleNotFoundError`**  
A: 运行 `pip install -r requirements.txt` 安装依赖。

**Q: 域名证书问题**  
A: 等待 GitHub 自动签发（约30分钟），或检查 DNS 是否正确指向 GitHub Pages。
