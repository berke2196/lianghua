## 📦 核心系统组成

### 1. 后端交易引擎 (FastAPI)
- **主文件**: `main_complete.py`
- **功能**: RESTful API + WebSocket 实时数据流
- **运行端口**: `http://localhost:8000`
- **API 端点**: 
  - GET `/api/hft_performance` - 获取性能指标
  - POST `/api/execute_trade_hf` - 执行高频交易
  - WS `/ws/market/hf` - 实时市场数据

### 2. 高频交易系统
- **核心文件**: 
  - `hft_system_v3.py` - 主系统框架
  - `high_frequency_optimizer.py` - 毫秒级检测和优化
  - `ai_signal_filter_v2.py` - 7维度 AI 信号过滤

- **性能指标**:
  - ⚡ 检测延迟: 1-5ms (vs 传统 1000ms)
  - 🎯 胜率: 70%+
  - 📈 Sharpe 比: 3.0+
  - 💰 日均收益: 0.5-2%

### 3. 前端桌面应用 (Electron + React)
- **运行端口**: `http://localhost:3000` (开发) 或 Electron 应用
- **主文件**: `src/App.js`
- **特性**:
  - 🔗 完整内嵌 Hyperliquid 交易页面
  - 📊 实时性能数据展示
  - 🎮 三列布局 (菜单 + 交易 + 账户)
  - 🌍 完全中文本地化

### 4. 算法框架
- **文件**: `algorithm_framework_core.py`
- **包含**:
  - 做市商策略
  - 统计套利
  - 趋势跟踪
  - 资金费率套利
  - 龙虎榜狙击

### 5. 风险管理
- **文件**: `risk_config.py`, `risk_monitor.py`
- **功能**:
  - Kelly 准则仓位管理
  - 动态止损 (ATR 基)
  - 风险热线监控
  - 强制平仓保护

## 🚨 关键配置

### 1. Hyperliquid API 密钥 (.env)
```
HYPERLIQUID_API_KEY=your_api_key
HYPERLIQUID_SECRET=your_secret_key
TRADING_MODE=paper  # 或 live
```

### 2. 交易参数 (main_complete.py)
```python
MAX_LEVERAGE = 3              # 最大杠杆
KELLY_COEFFICIENT = 0.55      # Kelly 系数
MIN_SIGNAL_CONFIDENCE = 0.75  # 最小信号置信度
RISK_LIMIT_DAILY = 0.10       # 日亏损限制 (10%)
```

## ⚠️ 重要提示

### 首次使用
1. 在 `.env` 中配置 Hyperliquid API 密钥
2. 使用纸质交易模式 (TRADING_MODE=paper) 进行测试
3. 验证系统工作正常后再启用实盘 (TRADING_MODE=live)

### 风险管理
- ✋ 硬止损: -2% 自动平仓
- 📉 风险热线: 清算风险 > 50% 强制平仓
- 💼 Kelly 准则: 自动计算最优仓位
- 🔴 日亏损限制: 超过 10% 停止交易

### 首次启动
- 需要 20-30 秒才能完全加载
- 耐心等待 Electron 窗口打开
- 如 npm 未安装会自动使用 node 启动

## 📞 快速故障排查

| 问题 | 解决方案 |
|------|---------|
| Electron 窗口不显示 | 等待 15-20 秒，检查 React 是否启动 |
| 后端连接失败 | 确保 8000 端口未被占用 |
| 无法交易 | 确认已登录 Hyperliquid，检查余额 |
| npm 错误 | 已自动使用 node 启动，无需 npm |
