## 算法框架 (60%)

| # | 算法名称 | 权重 | 日收益 | 特点 |

| 1️⃣ | 做市商 | 30% | 0.3-0.8% | ⭐⭐⭐ 最稳定 |

### 每个算法的工作方式

#### 1️⃣ 做市商算法 - 稳定收益

```
目标: 赚取买卖点差
原理: 
  1. 持续挂买单(低) + 卖单(高)
  2. 赚取中间的点差
  3. 超30秒无利润立即平仓
  
实例: BTC 价格 $45000
  - 买单: $44999.5 (赚 $0.50)
  - 卖单: $45000.5 (赚 $0.50)
  - 成本: $1 / 交易对 (平均)
  
日收益: 0.3-0.8%
特点: 高频、低风险、稳定
```

#### 2️⃣ 统计套利 - 低风险对冲

```
目标: 找两个币种的价格偏离
原理:
  1. 监测 BTC/ETH 比率
  2. 计算历史平均 (过去1000根K线)
  3. 当偏离超过2.5倍标准差时交易
  4. 等待恢复到均值时平仓
  
例子: BTC/ETH 历史比率 = 15
  - 当前比率 = 18 (BTC相对贵)
  - 信号: 卖 BTC, 买 ETH
  - 期望: BTC/ETH 回到 15
  
日收益: 0.1-0.5%
胜率: 70%+ (配对平衡)
```

#### 3️⃣ 趋势跟踪 - 大波段收益

```
目标: 识别和跟踪强势趋势
原理:
  1. 检查3个时间框都是上升/下降
     - 短期 (当前价 > SMA10)
     - 中期 (SMA10 > SMA30)  
     - 长期 (当前价 > 中点)
  2. 只有全部一致才下单
  3. 根据 MACD 强度决定下单量
  
例子: BTC 上升趋势
  - 短期: UP ✅
  - 中期: UP ✅
  - 长期: UP ✅
  → 信号强度: 75 → 做多
  
日收益: 0.5-2% (当有大趋势)
特点: 减少假信号, 抓大波段
```

#### 4️⃣ 资金费率套利 - 无风险

```
目标: 收取期货资金费率
原理:
  1. Hyperliquid 每8小时结算一次费率
  2. 如果费率 > 0, 多头支付空头
  3. 执行: 现货买 + 期货卖 (对冲)
  4. 收取资金费率
  
例子: 资金费率 0.1% / 8h
  - 年化收益 = 0.1% × 3 × 365 = 10.95%
  - 对于 $100万: 年赚 $109,500
  
年收益: 10-50%
胜率: 99% (完全对冲)
风险: 0 (对冲)
```

#### 5️⃣ 技术指标策略 - 辅助确认

```
使用的指标:
1. RSI (相对强弱): <30 买, >70 卖
2. MACD (动量): 金叉买, 死叉卖
3. 布林带: 突破上限卖, 突破下限买
4. 成交量: 量价确认

日收益: 0.2-0.6%
特点: 传统可靠, 用于确认其他信号
```

## AI 过滤层 (40%)

### AI 做什么?

从 100 条原始信号 → 10-20 条执行信号

```
原始信号流 (100/秒)
    ↓
AI 质量评分
AI 市场环境判断
AI 虚假信号检测
AI 执行时机优化
    ↓
高质量信号 (10-20/秒)
    ↓
风控检查
    ↓
下单执行
```

### AI 过滤的4个维度

#### 1️⃣ 信号质量评分 (0-100)

```
输入信号评估:
- 信号强度 (0-100)
- 信号类型 (做市商最优信)
- 方向明确性 (80 > 20)
- 重复性 (多次出现权重高)

输出: 质量分数
```

#### 2️⃣ 市场环境判断

```
检测当前市场状态:
- "trending": 趋势市 (适合趋势跟踪)
- "ranging": 区间震荡 (适合做市商)
- "volatile": 高波动 (谨慎)
- "sideways": 横盘 (等待)

根据环境调整信号权重
```

#### 3️⃣ 虚假信号检测

```
识别虚假信号 (避免追高杀低):
- 价格快速反向 (>2%) 
- 成交量异常低
- 技术指标背离
- 新闻事件冲击

删除虚假信号
```

#### 4️⃣ 执行时机优化

```
决定何时下单:
- 立即执行 (最佳点位)
- 延迟 100ms (确认)
- 延迟 500ms (等待)
- 不执行 (信号质量差)

最大化成交概率
```

### AI 使用的模型

```
1. Random Forest (信号分类)
   - 100棵树
   - 准确率: 75%
   
2. Gradient Boosting (市场判断)
   - 学习率: 0.05
   - 准确率: 82%
   
3. LSTM (虚假信号检测)
   - 时间步: 50
   - 准确率: 88%
   
4. XGBoost (执行时机)
   - 准确率: 80%
```

## 🔄 完整交易流程 (1秒循环)

```
[1] 数据获取 (1ms)
    获取: 实时价格, 成交量, 资金费率, 订单簿
    
[2] 5个算法并行运行 (50ms)
    做市商 → Signal 1
    统计套利 → Signal 2
    趋势跟踪 → Signal 3
    资金费率 → Signal 4
    技术指标 → Signal 5
    
[3] 信号融合 (20ms)
    计算加权平均: 
    = 0.30×S1 + 0.25×S2 + 0.25×S3 + 0.10×S4 + 0.10×S5
    
[4] AI 过滤 (50ms)
    质量评分
    市场判断
    虚假检测
    时机优化
    → 最终得分 (0-100)
    
[5] 风控检查 (10ms)
    检查头寸大小
    检查账户余额
    检查热线风险
    检查日/月亏损
    
[6] 执行决策 (5ms)
    如果得分 > 65 且通过风控:
    ✅ 下单执行
    否则:
    ❌ 观望
    
[7] 监控记录 (10ms)
    记录所有数据 (用于优化)
```
